## Memory Backend Decision

**Decision:** Use built-in SQLite backend with Gemini Embeddings 2 (not QMD)

**Rationale:**
- QMD requires local CPU for embeddings (embeddinggemma-300M model)
- VPS has no GPU, so QMD would be slow
- Built-in backend + Gemini Embeddings 2 allows cloud-based embeddings
- No local CPU/GPU load for embeddings

**Configuration:**
```json
{
  "agents": {
    "defaults": {
      "memorySearch": {
        "provider": "gemini",
        "model": "gemini-embedding-2-preview",
        "outputDimensionality": 1536,
        "remote": {
          "apiKey": "${GEMINI_API_KEY}"
        }
      }
    }
  }
}
```

**Key Details:**
- Gemini Embeddings 2 announced March 10, 2026 (latest from Google)
- Supports 768, 1536, 3072 dimensions
- **Recommended: 1536** (best quality-to-cost ratio)
- 3072 is default but 1536 achieves same quality per Google's benchmarks
- SQLite backend works out of box, no extra install
- Multimodal (images/audio) only works with QMD, not SQLite
