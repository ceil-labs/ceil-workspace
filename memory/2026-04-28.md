## 2026-04-28 00:00 — True BM25 Search with pg_textsearch

**Victor's ask:** Build true BM25 keyword search alongside semantic vector search.

### Problem
PostgreSQL's `ts_rank_cd` is NOT BM25. It's cover density ranking that rewards proximity and coverage, not BM25's term frequency saturation (k1 parameter) and length normalization (b parameter).

### Solution: Custom PostgreSQL Docker Image

**Dockerfile.postgres:**
- Base: `pgvector/pgvector:pg17` (includes pgvector extension)
- Downloads pg_textsearch v1.1.0 pre-built `.deb` from GitHub releases
- Installs via `dpkg -i`
- Adds `pg_textsearch` to `shared_preload_libraries` in postgresql.conf.sample
- Extension survives container restarts (unlike previous attempt where it was lost after restart)

**Migration: `migrations/003_pg_textsearch.sql`**
```sql
CREATE EXTENSION IF NOT EXISTS pg_textsearch;

CREATE INDEX IF NOT EXISTS idx_chunks_bm25 
ON chunks 
USING bm25(content, search_vector) 
WITH (text_config='english', k1=1.20, b=0.75);
```

### Search Modes

Three modes implemented in `src/routes/search.ts`:

1. **Vector (Semantic)** — `c.embedding <=> query_embedding`, cosine similarity
2. **FTS (BM25)** — `c.content <@> to_bm25query(query, 'idx_chunks_bm25')`
   - Returns ABS(bm25_score) because `<@>` returns negative scores
   - True BM25 with k1=1.20, b=0.75
3. **Hybrid** — Weighted combination:
   ```
   hybrid = ftsWeight * (bm25 / (bm25 + 1)) + (1 - ftsWeight) * cosine_similarity
   ```
   - BM25 normalized to [0,1] via `bm25 / (bm25 + 1)`
   - Vector similarity already in [0,1]
   - `ftsWeight` param controls balance (default 0.5)

### BM25 Behavior Discovery

**Query:** `"protek telecoms support"` returned pg_textsearch GitHub docs instead of resume.

**Why (correct behavior, surprising result):**
| Source | "support" occurrences |
|--------|----------------------|
| pg_textsearch GitHub docs | 12 times |
| Resume (Protek Telecoms) | 2 times |

BM25 treats query as separate tokens. "support" is a common word with low IDF, but high term frequency (12×) in technical docs boosts their score enough to outweigh the rare terms "protek" and "telecoms".

**Takeaway:** BM25 doesn't understand proper noun phrases. For phrase-sensitive search, Hybrid mode is better — semantic search understands "Protek Telecoms Support" as a company name concept.
