```
    ├── Automated: BloodHound, PowerView, WinPEAS, SharpUp
    └── HTB examples from Certified, Administrator, Cicada
```

All articles cite chunks from the ingested HTB sources. Victor wants to review before expanding further.

**Status:** PAUSED pending Victor's review.

**Update:** Victor reviewed the graph and noted the `see-also` cross-references between Enumeration and its children were redundant — the parent-child tree relationship already visually connects them in the graph. Removed both redundant `see-also` links. Cross-references should be reserved for non-hierarchical relationships (e.g., Linux Enumeration Tools → Privilege Escalation).

**Additional finding:** Multi-linkType per cross-reference entry is NOT allowed — `linkType` is a single string per entry. To express multiple relationship types between the same two articles, you need separate entries. This constraint is not documented in the skill.

**Feedback drafted:** `memoria-skill-feedback.md` for dev team — covers tree vs. links redundancy, multi-linkType constraint, and under-specified linkType semantics.
