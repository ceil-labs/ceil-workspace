## Cross-Reference Guidance Correction

**Victor's feedback:** Cross-references between wikis don't need to be bidirectional. They should be directional based on relationship semantics.

**Updated in all guidance docs:**
- **AGENTS.md** — "Cross-references are directional, not bidirectional by default. Only add a reverse link if the relationship semantics differ."
- **MEMORY.md** — Same correction
- **Memoria SKILL.md** — "Cross-references are typed and directional. Only add a reverse link if the relationship semantics differ — bidirectional links are not required."

**Example:**
- `supply-chain-security` → `tanstack-npm-supply-chain-attack` (see-also: "here's a real example")
- This is sufficient. No need for a reverse `see-also` unless the TanStack article wants to say "read the overview first" (which would be `prerequisite`, not `see-also`).
