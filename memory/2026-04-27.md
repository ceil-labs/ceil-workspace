## 2026-04-27 09:40 — Bug Fixes Discovered Through Testing

### 1. extractor.ts — Buffer vs String
**Problem:** Multer's `memoryStorage` returns `Buffer`, but `extractText` and `extractMarkdown` called `.trim()` and `.replace()` (string methods).
**Fix:** Changed to `content.toString()` before processing.

### 2. init.sql — IVFFlat Index on Small Tables
**Problem:** `CREATE INDEX idx_chunks_embedding ON chunks USING ivfflat (...)` returns 0 rows on small tables.
**Fix:** Removed the index from init.sql (already fixed manually in production earlier).

### 3. Routes — Invalid UUIDs Caused 500
**Problem:** `/sources/bad-uuid` hit DB with malformed UUID → 500 error instead of 404.
**Fix:** Added `isValidUUID()` guard to `sources.ts` and `chunks.ts` routes, returns 404 early.

### 4. store.ts — DATA_DIR Captured at Module Load
**Problem:** `const DATA_DIR = process.env.DATA_DIR || "./data"` evaluated once at import time, before test env was set.
**Fix:** Made DATA_DIR evaluation lazy (function call per use).

### 5. chunker.test.ts — Test Text Too Small
**Problem:** Test used single-paragraph text that didn't exceed chunk size, so no split occurred.
**Fix:** Updated test to use multi-paragraph text with enough content to force splitting.

**Result:** All 28 tests passing across 4 test files.
