### Key Finding from @chrysb Tweet

OpenClaw 3.11 (released Mar 12) introduced stricter "cron-owned delivery" contract:
> "if your jobs were set to delivery.mode='none' but still sending via message tool, they may now go silent"

However, this doesn't fully explain our case since:
- We were using `announce` mode (not `none`) when failures started
- Reports worked for 5+ days after 3.11 release
- Issue started after Mar 27 restart, not immediately after 3.11
