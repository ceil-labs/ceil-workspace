## 🎯 Why Production Doesn't Need It

**Production uses `bundle install`:**
```dockerfile
COPY Gemfile Gemfile.lock ./
RUN bundle install
```

Once you run `rails new .` in **development**, it generates a `Gemfile` with Rails as a dependency. When you build the **production** image later:

1. `COPY Gemfile Gemfile.lock` brings in the dependencies
2. `bundle install` installs Rails (and all other gems) automatically
3. No need for `gem install rails` — bundler handles it
