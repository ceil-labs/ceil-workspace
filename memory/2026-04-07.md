# 2026-04-07 — Dreaming Activated

## Summary
Dreaming feature is now fully activated via slash command. Previous configuration-only approach didn't create the cron job; `/dreaming on` was required.

## Dreaming Activation

### What Was Missing
- Config was set in `openclaw.json` (plugins.entries.memory-core.config.dreaming)
- Gateway was restarted
- **But**: No cron job was auto-created from config alone

### Solution
Used slash command interface to enable:
```
/dreaming on
```

### Confirmed Status
```
Dreaming status:

• sweep cadence: 0 3 * * * (3:00 AM daily)
• promotion policy: score>=0.8, recalls>=3, uniqueQueries>=3

Phases:

• DREAMS.md for human-readable summaries
```

## Key Learning

**Config ≠ Activation**

The `dreaming.enabled: true` in config is a **default**, but actual enablement requires:
1. `/dreaming on` — Activates cron job
2. `/dreaming off` — Deactivates
3. `/dreaming status` — Check current state

This is similar to how `session.reset` config sets defaults, but actual resets happen via commands.

## Research: Why Dreaming Cron Job Wasn't Created

### Issue Found
- Config was set correctly in `openclaw.json`
- `/dreaming on` enabled status to "on"
- But **no cron job was actually created** in `~/.openclaw/cron/jobs.json`
- This is a **bug in OpenClaw 2026.4.5** — the slash command updates in-memory status but doesn't persist the cron job
