### GitHub Issue #53747 — Kimi Coding Malformed Tool Calls

**Title:** Regression: Kimi Coding emits malformed read tool calls

**Summary:** OpenClaw 2026.3.23-1 with kimi-coding provider (k2p5 model) produces malformed tool calls — read/exec calls with missing path/command parameters.

**Symptom:** Repeated errors like:
- `read tool called without path`
- `Missing required parameter: path (path or file_path)`

**Root Cause:** Bug in OpenClaw's Kimi tool call argument parsing/repair logic.

**Reference:** https://github.com/openclaw/openclaw/issues/53747

### GitHub PR #54448 — The Fix

**Title:** preserve Kimi tool call arguments instead of clearing valid JSON

**Fixes:** Issue #53747 (and #54442, #54366)

**Changes:**
1. `tryParseMalformedToolCallArguments` — Return parsed args when JSON.parse succeeds (was incorrectly returning undefined for valid JSON)
2. **Remove `clearToolCallArgumentsInMessage`** — This was assigning `{}` to arguments, causing "must have required property" errors
3. `wrapStreamRepairMalformedToolCallArguments` — Stop clearing correctly-set arguments

**Reference:** https://github.com/openclaw/openclaw/pull/54448

**Status:** PR merged — should be in next OpenClaw build
